# 5/11/2026 Morning Upgrade

## Summary

Implemented a hybrid retrieval refactor plus multi-turn conversation context support for smoother RAG flows.

## Changes

- `src/core/retrieve.py`
  - Integrated `HybridSearchConfig` with default RRF behavior.
  - Updated hybrid retrieval to use `k=60` and fuse vector + BM25 evidence.
  - Preserved BM25 and vector scores during retrieval to build a better candidate ranking.
  - Added a robust hybrid ranking path before reranking, with fallback to top fused results.

- `src/core/retrieval_service.py`
  - Updated default hybrid search window to `k=60` per query.

- `src/core/brain_service.py`
  - Adjusted `Brain.search` to use the new retrieval tuning and a stronger rerank limit.

- `src/ui/app.py`
  - Added `ConversationContext` support for multi-turn queries.
  - Enhanced search queries with follow-up detection and recent session summaries.
  - Improved user feedback in the Streamlit status flow.

## Result

The system now performs stronger RRF-based hybrid retrieval and carries session-aware context into search, enabling more stable follow-up question handling in multi-turn RAG scenarios.
