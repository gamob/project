🦖 Project: Corporate Brain v1.0 (Gremlin RAG)
This is a local, air-gapped RAG (Retrieval-Augmented Generation) system built with Streamlit and LangChain, designed for secure, offline document querying.

Core Architecture:

The Orchestrator (brain_service.py): Acts as the central service managing the "brain" (the indices). It ensures the FAISS vector store and BM25 retriever are loaded into memory and ready for interaction.

Hybrid Retrieval (retrieve.py): The heart of the intelligence. It performs hybrid search (FAISS for semantic similarity + BM25 for keyword matching). It handles deduplication and uses a CrossEncoder reranker (bge-reranker-v2-m3) to score and refine the relevance of retrieved docs before passing them to the LLM.

Generation Engine (generate.py): Connects to a local Ollama instance (Llama 3.1). It manages context window budgeting (truncating context to ~8k chars), conversation history, and handles the prompt construction/streaming logic.

Embedding & Indexing (embed_store.py): Handles the vector space. Uses bge-m3 embeddings. It implements a strict "offline-first" mode and manages saving/loading local indices.

Data Pipeline:

Loading (load_data.py): A flexible loader that supports various formats (PDF, TXT, MD, DOCX, PPTX, XLSX, CSV) via a mapping of file extensions to LangChain loaders.

Processing (split_text.py): Implements specialized "chunking profiles" based on file types. It filters out tiny "noise" fragments (<80 chars) and enriches metadata for each chunk.

UI & Infrastructure:

Frontend (app.py): A Streamlit-based UI. It visualizes retrieval confidence scores (High/Medium/Low), handles source citations, and manages the chat session state.

Persistence (chat_store.py): A SQLite database implementation that stores chat history, session titles, and message logs persistently.

Configuration (config_service.py & config.json): Centralizes all paths and environment variables (model paths, index paths, etc.), ensuring the project root is always dynamically detected.

Key Technical Features:

Strict Offline Mode: No external API calls; everything runs locally (HF_HUB_OFFLINE, TRANSFORMERS_OFFLINE).

Confidence Logic: Calculates a confidence score based on the retrieval process to warn the user if the "Brain" isn't sure about the answer.

Hybrid Vietnamese/English Support: The system is primed to handle queries in both languages as needed by the user.
